# Licensing

Theatrical uses a split license model.

## MIT — core packages (maximum adoption)

These are free for any use, including commercial:

| Package | License |
|---------|---------|
| `@theatrical/sdk` | MIT |
| `@theatrical/cli` | MIT |
| `Theatrical.Sdk` (C#) | MIT |
| `theatrical` (Python) | MIT |

See [`LICENSE`](LICENSE).

## BSL 1.1 — premium packages (commercial protection, converts to MIT)

These are free for development, testing, and non-competitive production use.
Using them to offer a competing commercial SaaS/PaaS/IaaS product requires a
commercial license. Each automatically converts to the MIT License on its
Change Date.

| Package | License | Change Date | Converts to |
|---------|---------|-------------|-------------|
| `@theatrical/react` | BSL 1.1 | 2029-05-03 | MIT |
| `@theatrical/events` | BSL 1.1 | 2029-05-03 | MIT |
| `@theatrical/analytics` | BSL 1.1 | 2029-05-03 | MIT |
| `@theatrical/templates` | BSL 1.1 | 2029-05-03 | MIT |

The full BSL text is in [`LICENSE-BSL.md`](LICENSE-BSL.md) and ships inside each
premium package as its `LICENSE` file.

For alternative/commercial licensing: https://theatrical.dev/licensing
