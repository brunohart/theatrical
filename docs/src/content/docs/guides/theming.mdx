`@theatrical/react` ships a warm parchment design system as its default — warm backgrounds, burnt orange accents, deep navy structure. Every visual value is a token in `TheatricalThemeProvider`. This guide shows how to adapt those tokens to your brand without touching component internals.

## How the token system works

`TheatricalThemeProvider` accepts a `theme` prop containing partial token overrides. Tokens are organised into namespaces (`colors`, `typography`, `spacing`, `radii`, `shadows`, `transitions`). A partial override **merges within each namespace** — passing `{ colors: { accent: '#e60026' } }` changes only the accent color; all other color tokens keep their default values.

```tsx
import { TheatricalThemeProvider } from '@theatrical/react';

export function App() {
  return (
    <TheatricalThemeProvider theme={{ colors: { accent: '#e60026' } }}>
      <YourApp />
    </TheatricalThemeProvider>
  );
}
```

## Color tokens

The full color palette with their default values:

| Token | Default | Role |
|-------|---------|------|
| `colors.accent` | `#D4622B` | Burnt orange — primary CTA, selection highlight, brand signature |
| `colors.accentMuted` | `#B8532A` | Muted accent for hover states |
| `colors.bg` | `#F0EDE6` | Page background — warm parchment |
| `colors.surface` | `#F5F0E1` | Card and panel backgrounds — cream |
| `colors.surfaceRaised` | `#EBE6DA` | Interactive surface hover state |
| `colors.border` | `#D6D0C4` | Subtle dividers and outlines |
| `colors.text` | `#1A1A1A` | Primary text — ink, never pure black |
| `colors.textMuted` | `#8A8578` | Secondary text, labels, metadata — warm grey |
| `colors.textDisabled` | `#B8B3A8` | Disabled state text |
| `colors.success` | `#22c55e` | Confirmation states |
| `colors.warning` | `#D4622B` | Caution states |
| `colors.error` | `#C4391D` | Error and destructive states — rust |

## Typography tokens

| Token | Default | Description |
|-------|---------|-------------|
| `typography.fontFamily` | `'Inter, -apple-system, sans-serif'` | Body and UI text — clean, stays out of the way |
| `typography.fontFamilyHeading` | `'Space Grotesk, -apple-system, sans-serif'` | Headlines — tight tracking, poster energy |
| `typography.fontFamilyMono` | `'JetBrains Mono, monospace'` | Labels, metadata, reference codes — archival, systematic |
| `typography.fontSizeBase` | `'16px'` | Root font size |
| `typography.lineHeightBase` | `'1.5'` | Body line height |

The default type stack pairs Space Grotesk for compressed, confident headings with Inter for clean body text and JetBrains Mono for systematic labels:

```tsx
// Override with your own typeface pair
const theme = {
  typography: {
    fontFamily: '"DM Sans", system-ui, sans-serif',
    fontFamilyHeading: '"DM Sans", system-ui, sans-serif',
  },
};
```

<Aside type="tip">
Headlines should feel like poster type — compress them, track them tight. Mono labels work well for section markers, dates, and reference codes to create an archival, systematic voice.
</Aside>

## Spacing and radii

The spacing scale uses `rem` values. All component layout is derived from `spacing.base`:

| Token | Default |
|-------|---------|
| `spacing.xs` | `0.25rem` |
| `spacing.sm` | `0.5rem` |
| `spacing.md` | `1rem` |
| `spacing.lg` | `1.5rem` |
| `spacing.xl` | `2rem` |
| `spacing.xxl` | `3rem` |

Border radii control component roundness:

| Token | Default | Used by |
|-------|---------|---------|
| `radii.sm` | `4px` | Badges, chips |
| `radii.md` | `8px` | Cards, buttons |
| `radii.lg` | `12px` | Modals, panels |
| `radii.full` | `9999px` | Pill buttons, avatars |

For a sharper, more geometric look (common in European arthouse cinemas):

```tsx
const theme = {
  radii: {
    sm: '2px',
    md: '4px',
    lg: '6px',
    full: '4px',  // even pill buttons become slightly squared
  },
};
```

## Applying a complete brand theme

This example wires a full custom theme in the `react-ticketing` template. Create `src/theme.ts`:

```typescript
import type { DeepPartialTokens } from '@theatrical/react';

export const brandTheme: DeepPartialTokens = {
  colors: {
    accent: '#C4391D',
    accentMuted: '#A83018',
    bg: '#F0EDE6',
    surface: '#F5F0E1',
    border: '#D6D0C4',
  },
  typography: {
    fontFamilyHeading: '"Space Grotesk", system-ui, sans-serif',
  },
  radii: {
    md: '4px',
    lg: '8px',
  },
};
```

Then pass it to the provider in `src/App.tsx`:

```tsx
import { brandTheme } from './theme';

export default function App() {
  return (
    <TheatricalThemeProvider theme={brandTheme}>
      {/* ... */}
    </TheatricalThemeProvider>
  );
}
```

## Reading tokens in your own components

The theme context is available via `useTheatricalTheme()`:

```tsx
import { useTheatricalTheme } from '@theatrical/react';

function PriceTag({ amount }: { amount: number }) {
  const { colors } = useTheatricalTheme();
  return (
    <span style={{ color: colors.accent, fontWeight: 700 }}>
      {new Intl.NumberFormat('en-NZ', { style: 'currency', currency: 'NZD' }).format(amount)}
    </span>
  );
}
```

This keeps custom components visually consistent with the token system — if you change the accent color in the provider, your component updates automatically.

<Aside>
`useTheatricalTheme()` must be called inside a component tree wrapped by `TheatricalThemeProvider`. It throws if called outside the provider.
</Aside>
